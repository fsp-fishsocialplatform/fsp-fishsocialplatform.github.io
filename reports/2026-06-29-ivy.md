## 一、AI 识鱼服务重构：Dify → LangGraph

延续 06-27 诊断出的问题（confidence 硬编码、无纠错通路、dify 未真正验证），今天把 ai-service 整体重构。

**架构调整**
- 移除 Dify 工作流依赖，改为自建 LangGraph agent（`graph.py`）
- 新增 LLM provider 抽象层（`providers.py`）：支持 Anthropic 格式和 OpenAI 格式，通过环境变量 `LLM_BASE_URL` / `LLM_API_KEY` / `LLM_FORMAT` 切换，兼容自定义中转站
- LangGraph graph 内置自我纠错 guardrails：识别结果置信度不足时自动重试
- 新增 `fish_lookup` tool + fish template snapshot，让模型有结构化的鱼种知识库可查
- identify 接口 contract 新增 `species_id` 字段，打通「识鱼结果 → 渔场入缸」链路
- 部署到 Railway，Langfuse 接入做 LLM 调用追踪

**调试过程**（20:41–22:24 的一串 debug 提交）
- `02a40e9`：修正 Anthropic provider 的 path（`/v1/messages`）
- `b5d0b89`：OSS 签名 URL 无法被 Anthropic API 直接访问，改为下载图片后以 base64 发送
- 其余 debug 提交：补充日志、排查环境变量、修 `__pycache__` gitignore

**现状**：LangGraph graph 已在 Railway 跑通，Claude Sonnet 4.6（via 中转站）成功返回识鱼结果，Langfuse 追踪可见。

---

**总结**：AI 识鱼完成架构升级，从 Dify 黑盒换成自建 LangGraph agent，provider 抽象支持自定义中转，图片改 base64 传输后端到端跑通。本周的 AI 识鱼问题诊断（06-27）→ 架构设计（06-29 上午）→ 实装调通（06-29 晚）形成完整闭环。
