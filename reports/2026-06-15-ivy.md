> 今日新增 **fsp-admin** 管理后台仓库

## 一、新增管理后台 fsp-admin（前后端）

今天搭起了独立的运营管理后台。

**前端（fsp-admin，全新仓库）**
- 初始化项目：基础结构、配置（.env / .gitignore / package.json / TypeScript）
- 实现核心模块：路由、鉴权、API client，管理用户与帖子
- 初始页面：登录、Dashboard、内容管理
- 配置 Vite + GitHub Pages 部署，改用 HashRouter（避免静态托管深链 404），base path 设为 `/fsp-admin/`
- CI 改为从 release 分支部署，与其他仓库对齐

**后端**
- 实现 admin 管理功能，增强 user 模型

## 二、核心功能：多图发布与浏览（前后端打通）

把「动态」从单图升级为多图体验。

**移动端**
- DynamicPostManagement 支持多图上传与帖子管理
- DynamicContent 模型 + 详情页支持多图展示
- 详情页接入图片查看器（photo viewer），支持点击大图浏览
- 上传前集成图片压缩，减小流量与体积

**后端**
- 媒体上传/获取支持缩略图（thumbnail）
- 新增 GetMediaByIDs 批量获取媒体接口
- GetPost / Feed 响应补充解析后的媒体 URL

## 三、帖子管理与个人内容

- 实现帖子删除功能，DynamicContent 增加 authorId 标识作者
- 后端实现帖子软删除（soft delete）
- 移动端接入 MyPostsStore 管理用户自己的帖子
- Feed 列表排除用户本人的帖子

## 四、性能与体验优化

- 后端新增 Server-Timing 响应头，跟踪接口延迟
- 移动端 PerformanceInterceptor 记录服务端耗时，便于定位慢接口
- Feed 改为 keyset 分页（替代 offset），重构动态流加载与分页逻辑
- DynamicTab / ProfileTab 在 initState 中延迟状态更新，避免构建期报错
- 调整 Gradle 属性提升构建性能
- 调整 healthcheck 超时与 GORM logger 设置，改善迁移性能

## 五、基础设施

- 集成 Firebase（性能监控 + Crashlytics 崩溃上报），debug 模式下开启验证
- Android CI 支持构建 release APK 并发布到 GitHub Release，新增 changelog 自动生成
- 新增 cached_network_image、sqflite 依赖
- 前后端各加 merge-to-release 一键发版脚本

---

**总结**：今天最大的进展是新建了 **fsp-admin 运营管理后台**（React + Vite，含鉴权/用户/内容管理，已配 GitHub Pages 部署），后端同步实现 admin 管理接口。同时完成「动态多图」前后端全链路（上传压缩 → 缩略图 → 多图浏览），落地帖子软删除/删除、Feed keyset 分页，接入 Firebase 监控与 release APK 自动发布（含 changelog）。
