## 一、地图标记重构：从悬浮 Widget 到原生聚合 Marker

此前地图发现页用 Flutter 悬浮 Widget（`Positioned` + `Stack`）在地图上叠加鱼形标记，标记之间没有聚合，点位密集时会互相遮挡。这次重构改成用高德地图原生 `Marker`：

- 新增点聚合算法 `marker_clustering.dart`：纯 Dart 实现，无 Flutter widget / 插件依赖。本地 `amap_flutter_map` 插件没有屏幕投影 API（无 `toScreenLocation`），所以聚类不能依赖插件把经纬度投到屏幕像素，只能在 Web-Mercator 像素空间自算——按当前 zoom 把经纬度投到"世界像素"，再按固定像素网格分桶合并，标准 grid-based clustering 做法，跟高德/美团 web 端一致，点数量级（几十）下 O(n)。用统一抽象 `ClusterablePoint`（id/lat/lng）覆盖排行/渔场/大鱼机会/钓点/活动 5 种发现条目类型，聚类层不关心具体业务类型，点击时再按模式反查原始 item（推迟分叉到最末端消费方）。
- 新增 `cluster_marker_icon.dart`：用 `dart:ui` 的 `PictureRecorder` + `Canvas` 把聚合泡泡（蓝色圆底+白色数字）和单点鱼形直接画成 PNG 字节喂给原生 `Marker.icon`，按内容（数量档位/选中态）缓存避免重复光栅化。
- `map_tab_screen.dart` 接入相机缩放/平移回调 `onCameraMoveEnd`，缩放级别变化时重新聚合重建 marker 集合；加了重建序号防止异步光栅化过程中缩放已变化导致的竞态。

## 二、渔场详情页：异步加载 + 地图导航

- `FisheryDetailScreen` 改成异步加载渔场数据（接入 `DiscoveryRepository`），加载中显示骨架屏 `_FisheryDetailSkeleton`，查不到数据时弹提示并返回上一页；距离/标签/简介等字段为空时不再渲染空 chip。
- 原来的"发起比赛"按钮（对应"当前不在 MVP 内"的占位卡片）替换成"导航"按钮：唤起高德地图导航，优先拉起高德 APP（`amap://` deeplink），未安装则降级到高德网页版（`uri.amap.com`）；坐标统一按 GCJ-02（`dev=0`）传递。
- 原"第一期决策信息"占位卡片简化为"渔场信息"卡片，展示位置和访问量。

---

**总结**：延续这几天地图发现模块从 mock 悬浮标记走向可用产品的节奏。今天做的是两件相互独立又衔接的事——点聚合让密集点位不再互相遮挡，导航让"看到渔场"到"能去渔场"之间补上最后一环。聚类算法按纯 Dart 实现、不依赖插件屏幕投影 API 是这次的关键技术决策，也让算法本身可以脱离 Flutter widget 独立单测。
