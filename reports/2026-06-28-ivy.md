## 一、Agora RTC 真实语音接入

昨天（06-27）语音房间的麦克风层是 mock 骨架，今天完成真实 RTC 接入。

**移动端**
- 新增 `AgoraVoiceService`：封装 init / join / leave / mute 生命周期，入房时请求麦克风权限
- 新增 `AgoraConfig`：从 `.env.local` 读取 `AGORA_APP_ID`（用 `flutter_dotenv`，敏感配置不入库）
- `VoiceRoomScreen` 入场自动加入 Agora 频道，dispose 时离开；底部 mute 按钮实时切换麦克风状态（红/绿图标）
- Android Manifest 补充 `RECORD_AUDIO` + `BLUETOOTH` 权限
- 新增 `.env.local` 到 `.gitignore`，避免 App ID 泄漏（`2c5bbc4`）

## 二、Android 构建问题修复

接入 `agora_rtc_engine ^6.3.0` 后触发一连串 Android 构建兼容问题，逐一修掉：

- `0997797`：将 Android compileSdk 升至 36（agora 6.5.x 依赖要求）
- `e9fbb49` / `ad746c7`：用 `plugins.withId` 方式为插件子项目统一设置 compileSdk 36，避免 Gradle 配置顺序问题
- `60f2f17`：允许 agora_rtc_engine 6.5.x 的重复 AAR namespace（构建警告转为许可）
- `fbbb109`：修 WebSocket URL 拼接错误——`https://` 未转换为 `wss://`，导致 WS 连接失败

## 三、语音房间体验修复（凌晨）

- `8e71930`：键盘弹起时聊天输入框被遮挡——用 `MediaQuery.viewInsetsOf` 检测键盘，键盘出现时隐藏语音席位格，确保输入框始终可见

---

**总结**：今天把语音房间从「UI 骨架」推进到「真实可用」——Agora RTC 接入完成，mute/unmute 实时生效，App ID 通过 dotenv 管理不入库。附带修掉了 agora 6.5.x 带来的 Android compileSdk 和 AAR namespace 构建兼容问题，以及 WS URL 拼接 bug。延续 06-27 的比赛模块，语音对战链路现在前后端+RTC 全部打通。
