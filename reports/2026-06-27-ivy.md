## 一、比赛模块（竞技对战）前后端实装

延续渔场养鱼游戏 P1 完成后的下一条故事线：基于渔场发起实时钓鱼比赛，支持多人语音房间与排名直播。

**后端**

- 新增 `internal/competition` 包：竞赛房间 CRUD、选手加入/投票、WebSocket hub（实时排名 + 聊天广播）
- DB schema 通过 GORM AutoMigrate 自动建表：`competition_rooms`、`competition_participants`、`competition_chat`
- 依赖 `golang.org/x/net/websocket`，无新外部依赖
- 配套单元测试 `competition_test.go`（115 行）

**移动端**

- 新增 `LaunchCompetitionScreen`：发起比赛表单（规则、时长、名额等）
- 新增 `VoiceRoomScreen`：比赛直播间，含语音席位、实时排名榜、文字聊天栏（约 490 行）
- `CompetitionStore` 通过 WebSocket 订阅实时排名更新与聊天消息
- 语音层为 UI 骨架，麦克风状态目前 mock（RTC SDK 集成待后续）
- 入口：`FisheryDetailScreen` 「发起比赛」按钮 → `/map/create-match` → `/information/voice-room`，两个 stub 路由替换为真实页面
- 新增路由注册、`competition_models.dart`（185 行）、测试 `competition_models_test.dart`（169 行）

---

**总结**：今天把比赛/语音房间从零落地——后端 WebSocket hub + DB 建模，前端发起表单 + 语音直播间 UI，两端同步提交。语音通话层仍是 mock 骨架，RTC 接入是下一步。整体节奏上，本周完成了渔场游戏 P1（06-26）和比赛模块（06-27）两条主线，发现模块的数据层也已就绪。
